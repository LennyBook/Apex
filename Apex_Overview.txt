[Apex_System_Overview]
Description: High-Level Overview of Apex System
- Apex is a modular platform for managing projects (Len_Writing, Video_Games, Home_Household, Websites), with self-improvement mechanisms via rules, insights, and audits.
- Core capabilities: Task tracking (ATS), file standards (AFT), overview (AO), navigation (AH), session analysis (ASAP).
- Self-improvement: Learnings (AIS), rules (RC), tags (TC), audits (GAR).
- GitHub integration: Raw links for latest/stable fetches, bypassing workspace truncation.

[Design]
Description: Core Design Principles
- Modular structure with standalone masters (unversioned filenames, internal versions).
- Self-healing via SHL rules, validation (VAL).
- Scalability for multiple projects via hubs.

[Current_Capabilities]
Description: Key Features
- Task management with archiving (ACTA).
- File access via GitHub raw links for latest/stable.
- Session start with ASAP for analysis and improvements.
- Learning capture for continuous evolution.

[Learnings]
Description: Captured Insights
- INS001–INS072: From session logs, e.g., INS072: "Git workflow avoids manual renaming, tracks only key versions for latest/stable."
- AI limitations: Truncation in prompts; GitHub links as workaround.

[System_Evolution]
Description: Roadmap and Changes
- Shift to unversioned masters with Git tags for versioning.
- Deprecate workspace for GitHub fetches.
- Iterative audits and learnings drive improvements.
